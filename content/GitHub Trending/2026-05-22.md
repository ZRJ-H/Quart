## 🔍 深度分析

### 🏆 今日动量之王：colbymchenry/codegraph（+4,294⭐，46.8%）

**为什么增这么快？**

codegraph 解决了AI编码代理（Claude Code、Codex、Cursor等）的核心痛点：**每次对话都重新读取大量文件，token消耗巨大**。它通过预索引代码知识图谱，让代理「知道在哪里」而不必「读取所有内容」。

今日暴增4,294星（增长率46.8%！）的驱动因素：
1. **精准命中Claude Code生态爆发期** —— Anthropic官方插件目录今日同步上线（anthropics/claude-plugins-official），两大事件叠加形成「Claude Code工具链」热点
2. **立即可用的本地化方案** —— 完全本地运行，无需云端API，对企业级用户极具吸引力
3. **「减少工具调用」的量化收益** —— 项目文档中明确给出token节省数据，开发者用数据说话，传播力强

**为什么之前不为人知？** 项目名「codegraph」过于通用，且核心受众是「已在使用AI编码代理的开发者」——这个群体本身就在过去2个月才快速扩大。

### 🗺️ 值得关注的赛道

**1. Claude Code 插件/技能生态正在「机构化」**

今日Trending中，直接与Claude Code相关的项目占据5席：
- `anthropics/claude-plugins-official`（Anthropic官方插件目录，🆕首次入榜）
- `colbymchenry/codegraph`（代码知识图谱，🆕首次入榜）
- `multica-ai/andrej-karpathy-skills`（Karpathy背书技能包，连续在榜）
- `Imbad0202/academic-research-skills`（学术研究技能包，🆕首次入榜）
- `dotnet/skills`（.NET官方技能包，🆕首次入榜）

**判断**：Claude Code从一个「好用的终端工具」正在演变为「有插件标准、有技能市场、有官方目录」的平台级产品。这与VS Code的扩展市场路径高度相似。

**2. 终端AI代理进入「低星高增速」阶段**

`can1357/oh-my-pi`（5.8K⭐，+500/日）和 `truelockmc/streambert`（4.0K⭐，+1,094/日）代表一类新项目：它们不是「AI助手」，而是「用AI重新发明的终端/桌面工具」。这类项目总星不高，但增速惊人——说明早期采用者正在快速涌入。

**3. 本地优先（Local-first）AI工具持续升温**

`dotnet/skills`、`antoinezambelli/forge`、`rmyndharis/OpenWA` 都有一个共同点：**可自托管、无云端依赖**。在欧洲AI法案和美国数据主权讨论的背景下，「本地优先AI工具」从开发者偏好变为合规需求。
