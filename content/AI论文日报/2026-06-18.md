## 今日精选

### 1. CEO-Bench：AI Agent能玩长线游戏吗？

**原标题**：CEO-Bench: Can Agents Play the Long Game?
**机构**：Princeton University
**代码**：[❌ 未开源]
**链接**：[arXiv](https://arxiv.org/abs/2606.18543) | [HF Papers](https://huggingface.co/papers/2606.18543)

**核心贡献**：
该工作提出了 CEO-Bench，一个评估语言模型 Agent 在模拟创业环境中运营公司 500 天的长周期基准。Agent 需要同时处理定价、营销、预算等多维度 C-suite 决策，通过可编程 Python 接口与环境交互。基准重点考察四大能力：长周期不确定性导航、噪声环境下的信息获取、动态世界中的适应性、以及多线任务的协调编排。实验显示，仅有 Claude Opus 4.8 和 GPT-5.5 能勉强维持在 100 万美元起始资金以上，且无法稳定盈利；大多数前沿模型严重亏损。该基准首次系统衡量了 AI Agent 的持续性自适应智能。

**与AI Engineering的关联**：
对 Agent 工程化落地有直接指导意义——它暴露了当前 Agent 在长期记忆管理、多步推理一致性、资源分配等方面的系统性能短板，为 Agent 架构设计（如规划器、记忆模块、反思机制的迭代）提供了明确的方向指引。

### 2. 用图灵奖励学习用户模拟器

**原标题**：Learning User Simulators with Turing Rewards
**机构**：MIT
**代码**：[❌ 未开源]
**链接**：[arXiv](https://arxiv.org/abs/2606.19336) | [HF Papers](https://huggingface.co/papers/2606.19336)

**核心贡献**：
传统用户模拟器的训练目标是让模型模仿单一的 ground-truth 回复（通过对数似然或相似度奖励），但这种方式忽略了用户行为的多样性和上下文依赖性。本文提出 **Turing-RL**，一种基于图灵测试思想的强化学习方法：用 LLM 评委根据对话上下文历史，评估模拟器生成的回复是否无法与真实用户区分开来。该方法在对话聊天和 Reddit 论坛讨论两个场景下进行了验证，在 LLM 评估和人工评估两个维度上均显著优于基线方法。核心洞见是：不可区分性（indistinguishability）比逐回复精确匹配是更有效的训练目标。

**与AI Engineering的关联**：
对 Agent 评测流水线有重要价值——高质量用户模拟器可以替代昂贵的人工评估，大幅降低 RAG 对话系统、客服 Agent、以及多轮交互 LLM 应用的迭代测试成本。Turing-RL 的训练范式可直接应用于构建 Agent 回归测试所需的「虚拟用户」。

### 3. Sumi：从头训练的开放均匀扩散语言模型

**原标题**：Sumi: Open Uniform Diffusion Language Model from Scratch
**机构**：Tohoku University / RIKEN
**代码**：[✅ 开源]（权重 / 检查点 / 数据配方全部公开）
**链接**：[arXiv](https://arxiv.org/abs/2606.19005) | [HF Papers](https://huggingface.co/papers/2606.19005)

**核心贡献**：
Sumi（日语「墨」）是首个从头预训练的大规模均匀扩散语言模型（UDLM）。与自回归模型逐 token 生成或 Masked Diffusion 的分块生成不同，UDLM 允许在任意位置、任意步骤更新任意 token，天然支持双向上下文建模和非自回归生成。Sumi 参数量 7B，训练数据 1.5T tokens，全部使用公开语料。在知识、推理和编程基准上，Sumi 与同等 token 预算的自回归模型表现相当；在常识推理上稍弱，作者归因于训练数据中教育类文本占比较高。整套资产（权重、检查点、数据混合配方）完全公开。

**与AI Engineering的关联**：
为 LLM 推理优化提供了全新路径——扩散语言模型天然适合并行解码和可控生成，在低延迟推理、文本编辑、填空补全等场景中可能比自回归模型更具工程优势。作为完全开放的基座模型，可直接用于微调实验和架构对比研究。

### 4. SAE 干预不可靠：被抑制行为的后干预恢复

**原标题**：SAE Interventions are Unreliable: Post-Intervention Recovery of Suppressed Behavior
**机构**：The Hong Kong Polytechnic University（香港理工大学）
**代码**：[✅ 开源] [GitHub](https://github.com/Mingyuee88/sae-post-intervention-recovery) | [项目主页](https://mingyuee88.github.io/sae-post-intervention-recovery/)
**链接**：[arXiv](https://arxiv.org/abs/2606.18322) | [HF Papers](https://huggingface.co/papers/2606.18322)

**核心贡献**：
稀疏自编码器（SAE）被广泛用于将 LLM 残差流激活分解为可解释特征，近年来的潜空间安全防御也依赖于此——假设锁定某个「不安全」SAE 特征就能可靠阻止有害行为。本文揭示了一个严重的失效模式：即便成功钳制目标特征，模型仍可通过残差空间的约束优化绕过干预，恢复原有行为（称为「后干预恢复」）。作者将该问题形式化为一个受约束的残差空间优化问题，在更强的威胁模型下（干预在整个生成过程中持续激活），依然实现了行为恢复。尤其在安全关键场景的拒绝转向实验中，有效样本恢复率达 **95.8%**，而被防御特征的相对偏移仅 0.131。进一步的归因分析将恢复路径定位到 SAE 重构残差——即 SAE 未能解释的那部分信号。

**与AI Engineering的关联**：
对 LLM 安全对齐工程敲响警钟——仅依赖 SAE 特征级干预来保证模型行为安全是不可靠的，安全工程师需要搭配行为级护栏（如输出检测、多层级沙箱）才能形成纵深防御。同时也启发了更鲁棒的模型可解释性工具链设计。

### 5. Xcientist：将 AI 科学家的研究综合与验证外部化

**原标题**：Externalizing Research Synthesis and Validation in AI Scientists through a Research Harness
**机构**：Shanghai Jiao Tong University（上海交通大学）
**代码**：[❌ 未开源]
**链接**：[arXiv](https://arxiv.org/abs/2606.18874) | [HF Papers](https://huggingface.co/papers/2606.18874)

**核心贡献**：
当前 AI 驱动的科研工作流往往将推理链条隐藏在模型内部，难以审查和复现。Xcientist 提出了一个「研究夹具」（Research Harness），将 AI 科学家的完整研究流程——文献综合、想法生成、实验执行、结果验证——外部化为可审查的、受契约约束的制品（contract-governed artifacts）。论文发现了一个关键失败模式「声明漂移」（claim drift）：最终可运行的实验制品不再支持最初声称的机制，即 AI 科学家在过程中悄悄偏离了原始假设而未曾察觉。在训练无关的记忆系统、图结构交通预测、多尺度物理信息神经网络三个领域进行了验证。

**与AI Engineering的关联**：
对 AI Agent 在科研自动化中的工程落地有方法论价值——它本质上是一套 Agent 工作流的可审计性框架，其「契约约束」机制可以直接启发 LLM Agent 系统中的可追溯日志、中间结果校验和声明-证据链的设计，确保 Agent 的自主行为不被黑箱化。
