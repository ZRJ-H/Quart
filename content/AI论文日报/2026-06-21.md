## 今日精选

### 1. LedgerAgent：工具调用智能体的结构化状态管理
- **原标题**: LedgerAgent: Structured State for Policy-Adherent Tool-Calling Agents
- **作者/机构**: Md Nayem Uddin, Amir Saeidi, Eduardo Blanco, Chitta Baral（Arizona State University）| [arXiv](http://arxiv.org/abs/2606.20529)
- **核心贡献**: ① 解决工具调用智能体在多轮对话中隐式状态管理导致的决策错误和策略违规问题——智能体常基于过时或缺失的信息做决策，或工具调用违反领域策略 ② 提出 LedgerAgent，维护独立的账本（ledger）显式记录任务状态，每次决策前将状态注入提示词，并在执行环境变更的工具调用前检查策略约束以阻止违规 ③ 在 4 个客户服务领域和多种开源/闭源模型上，平均 pass@k 显著优于标准基于提示的工具调用方法，在严格的多轮一致性指标下优势最大
- **工程关联**: 对 Agent 开发中工具调用框架的状态管理设计有直接参考价值——分离状态存储与推理逻辑，可显著降低智能体在多轮交互中的"遗忘"和策略违规问题
- **价值点**: 关注点分离（状态账本 vs 推理提示）的架构设计，可直接复用到生产级 Agent 系统的状态管理层

### 2. 探针迭代调优：编码智能体的仓库指导文件自动优化
- **原标题**: Probe-and-Refine Tuning of Repository Guidance for Coding Agents
- **作者/机构**: Asa Shepard, Jeannie Albrecht | [arXiv](http://arxiv.org/abs/2606.20512)
- **核心贡献**: ① 解决 AGENTS.md 等仓库指导文件对编码智能体是否有效的争议——此前研究结论矛盾 ② 提出 probe-and-refine tuning 方法：用合成 bug-fix 探针迭代诊断指导文件的不足并自动修复，全程仅需单次 LLM 调用，无需 Agent 循环或工具调用 ③ 在 SWE-bench Verified 上，Qwen3.5-35B-A3B 的解决率从无指导基线的 25.5% 提升至 33.0%（p < 0.001），改善来源于覆盖率（多触及 14.5 个百分点实例）而非补丁精确度；指导文件使 Agent 能有效利用更大的步骤预算
- **工程关联**: 对使用 Coding Agent（Claude Code / Codex CLI / OpenCode 等）的团队有直接价值——提供数据驱动的仓库指导文件自动优化方法
- **价值点**: 全自动的指导文件迭代优化流程，可集成到 CI/CD 中持续提升 Coding Agent 的表征能力

### 3. 传染网络：多智能体 LLM 系统中的评估者偏差传播
- **原标题**: Contagion Networks: Evaluator Bias Propagation in Multi-Agent LLM Systems
- **作者/机构**: Zewen Liu | [arXiv](http://arxiv.org/abs/2606.20493)
- **核心贡献**: ① 形式化描述评估者偏差在多智能体 LLM 网络中的传播机制 ② 在 3 智能体实验中测量跨智能体传染矩阵，发现偏差持续传播（γ∈[0.157, 0.352]），同构模型传染系数仅为跨模型传播的 1/3–1/5，处于抑制区 ③ 将评估者委员会从 1 人增至 3 人可降低 72.4% 的有效传染——提供可操作的缓解策略，并开源实验框架
- **工程关联**: 对多智能体评审、辩论、协作系统的评估架构设计有直接指导——使用委员会机制和异构模型可有效抑制偏差级联
- **价值点**: 量化的偏差传播模型 + 委员会投票缓解方案，可直接指导多智能体评估系统的可靠性设计

### 4. UltraQuant：面向上下文密集型智能体的 4 比特 KV 缓存
- **原标题**: UltraQuant: 4-bit KV Caching for Context-Heavy Agents
- **作者/机构**: Inesh Chakrabarti, David Limpus, Aditi Ghai Rana et al. | [arXiv](http://arxiv.org/abs/2606.20474)
- **核心贡献**: ① 针对多轮 Agent 场景中长前缀复用导致的 KV 缓存压力问题，研究 4 比特 KV 缓存压缩 ② 提出 UltraQuant FP4 近似路径（FP8 查询 + FP4 KV 张量 + UE8M0 组缩放），结合 Walsh-Hadamard 旋转、不对称 K/V 处理等工程优化 ③ 在多轮 Agent 负载下，P50 首 token 延迟降低 3.47 倍（缓存压力轮次），全轮次加速 2.3 倍，输出吞吐提升 1.63 倍（AMD GPU 部署）
- **工程关联**: 对部署长上下文 Agent 服务的团队有直接价值——降低推理延迟和显存占用，提升并发服务能力
- **价值点**: AMD GPU 上的可复现量化工程方案，为 Agent 推理服务的成本优化提供完整的技术路径

### 5. H-RePlan：跨设备智能体系统的分层恢复框架
- **原标题**: Beyond Global Replanning: Hierarchical Recovery for Cross-Device Agent Systems
- **作者/机构**: Shu Yao, Yuhua Luo, Qian Long et al. | [arXiv](http://arxiv.org/abs/2606.20487)
- **核心贡献**: ① 解决多设备 Agent 系统中粗粒度恢复（重试/重分配/全局重规划）无法区分设备内可修复错误与需跨设备协调的瓶颈 ② 提出 H-RePlan 分层重规划框架，为每个设备配备可互换的执行策略，通过紧凑的跨层故障抽象分离设备本地恢复与编排器级全局重规划 ③ 在 HeraBench 跨设备故障注入基准（Linux + Android）上，完成率、指令遵循率和完全通过率大幅超越单策略和粗粒度基线，同时降低可靠端到端成功所需的 token 成本
- **工程关联**: 对跨平台/多设备 Agent 编排系统（如跨 Linux+Android 的 RPA 流程）的容错设计有直接参考
- **价值点**: 作用域感知的分层恢复架构，可推广到任何跨环境 Agent 系统的鲁棒性设计
