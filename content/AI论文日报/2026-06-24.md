## 今日精选

### 1. SelfCompact：让 Agent 自己决定何时压缩上下文
- **原标题**: Self-Compacting Language Model Agents
- **作者/机构**: Tianjian Li, Jingyu Zhang, William Jurayj, Xi Wang, Chuanyang Jin 等（Johns Hopkins University / Apple）| [arXiv](http://arxiv.org/abs/2606.23525)
- **核心贡献**: ① 解决长 Agent 轨迹（思维链 + 工具调用）积累过期内容导致上下文窗口溢出和后续生成被锚定在陈旧信息上的问题 ② 提出 SelfCompact，让模型通过压缩工具 + 轻量级触发规则自行决定何时压缩：子任务完成或轨迹收敛时触发，中间推导或卡住时抑制 ③ 在 6 个基准（数学竞赛 + Agent 搜索）和 7 个模型上，SelfCompact 匹配或超越固定阈值压缩方案，同时节省 30-70% token 成本；数学任务提升最多 18.1 分，Agent 搜索提升 5-9 分
- **工程关联**: 对任何使用长链工具调用的 Agent 系统（如 Coding Agent、RAG Agent）有直接价值——用极少规则替换固定阈值压缩，降低推理成本的同时提升准确率
- **价值点**: 无微调、无外部监督的自适应压缩机制，可直接集成到现有 Agent 框架的上下文管理模块

### 2. Tmax：终端智能体的最强开源 RL 训练方案
- **原标题**: Tmax: A simple recipe for terminal agents
- **作者/机构**: Hamish Ivison, Junjie Oscar Yin, Rulin Shao, Teng Xiao, Nathan Lambert, Hannaneh Hajishirzi（Allen AI / University of Washington）| [arXiv](http://arxiv.org/abs/2606.23321)
- **核心贡献**: ① 解决终端 Agent 领域缺乏开源 RL 训练基线和高质量训练数据的问题——此前学术界几乎未涉足 Agent 的 RL 训练 ② 提出 Taxonomy 驱动的数据生成方法（难度控制 + 角色画像 + 验证器多样化），产出一个比此前最大终端 Agent 数据集大 2.5 倍的训练集 ③ 仅用 9B 参数模型经 RL 训练后，在 Terminal-Bench 2.0 上达到 27% 解决率，超越此前更大模型；代码、数据、模型全部开源
- **工程关联**: 为终端 Agent（如 Claude Code、OpenCode、Codex CLI 等）的训练和微调提供了可复现的 RL 配方和高性价比的数据生成流水线
- **价值点**: 全开源的端到端方案（数据→训练→评估），团队可直接复现或基于此构建自己的终端 Agent 训练管线

### 3. EnterpriseClawBench：来自真实工作场景的 Agent 基准
- **原标题**: EnterpriseClawBench: Benchmarking Agents from Real Workplace Sessions
- **作者/机构**: Jincheng Zhong, Weizhi Wang, Che Jiang, Kai Tian 等（FrontisAI）| [arXiv](http://arxiv.org/abs/2606.23654)
- **核心贡献**: ① 解决现有 Agent 基准脱离真实企业工作场景的问题——Agent 在实际工作空间中处理异构文件、调用工具、交付业务制品的能力缺乏有效评估 ② 从真实企业会话存档中重建 852 个可复现任务，每个任务附带还原的环境装置、重写提示、角色类别、技能子类、硬约束和语义评分标准 ③ 最佳配置（Codex + GPT-5.5）仅达 0.663 分，表明企业级 Agent 仍有巨大提升空间；论文强调企业 Agent 评估应报告工具-模型组合、制成品质量、成本、运行时等多维指标，而非单一分数
- **工程关联**: 对在企业环境中部署 Agent 的团队有参考价值——其多维评估体系（不只是准确率）可直接指导 Agent 选型和性能监控
- **价值点**: 首个从真实企业工作流程中构建的 Agent 基准，评估维度贴近生产环境实际需求

### 4. Randomized YaRN：用随机位置编码突破长上下文推理
- **原标题**: Randomized YaRN Improves Length Generalization for Long-Context Reasoning
- **作者/机构**: Manas Mehta, Fangcong Yin, Greg Durrett（UT Austin）| [arXiv](http://arxiv.org/abs/2606.23687)
- **核心贡献**: ① 解决 LLM 在短序列上预训练后难以泛化到超长上下文推理的问题——现有扩展方案在超出训练长度后性能急剧下降 ② 提出 Randomized YaRN：在短上下文训练时，每个 token 从更大的位置范围中随机采样 YaRN 位置编码，并配合长度课程，使模型在短数据上就能接触分布外的位置表示 ③ 在 BABILong 和 MRCR 两个长上下文推理基准上，训练上下文 <8K 的条件下，Randomized YaRN 在 16K-128K 长度上持续超越标准微调，最大增益出现在远超训练分布的长度上
- **工程关联**: 对需要长上下文推理的 RAG 系统、文档分析、代码库理解等场景有直接价值——无需长上下文预训练即可获得泛化能力
- **价值点**: 将长上下文泛化能力从昂贵的大规模预训练中解耦，以低成本训练方案实现

### 5. 渐进式语言模型：将更多参数分配给浅层
- **原标题**: Tapered Language Models
- **作者/机构**: Reza Bayat, Ali Behrouz, Aaron Courville（Mila / University of Montreal）| [arXiv](http://arxiv.org/abs/2606.23670)
- **核心贡献**: ① 挑战语言模型各层均匀分配参数的默认设计——证据表明深层主要负责残差流的精炼而非变换，参数应反映这种不对称性 ② 提出 Tapered Language Model（TLM），通过余弦调度将 MLP 宽度从浅层向深层单调递减，固定总参数量下重新分配容量 ③ 在 3 个模型规模和 4 种架构（Transformer、Gated Attention、Hope-attention、Titans）上，锥形分配一致优于均匀基线，零额外参数和计算成本即可提升困惑度和下游基准性能
- **工程关联**: 为 LLM 架构设计提供了一个「免费午餐」级别的优化方向——不增加任何成本即可提升性能，可应用于新模型训练或架构改进
- **价值点**: 架构无关的设计原则，任何基于 Transformer 的模型家族均可受益，相当于在现有架构中发现了一个被忽视的优化杠杆
