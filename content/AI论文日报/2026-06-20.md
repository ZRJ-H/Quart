## 今日精选

### 1. OmniAgent：原生主动感知驱动的全模态视频理解智能体

**原标题**：Native Active Perception as Reasoning for Omni-Modal Understanding
**作者/机构**：Harry Hsing 等（多机构合作）
**链接**：[arXiv](http://arxiv.org/abs/2606.19341) · [GitHub](https://github.com/harryhsing/OmniAgent) · ICML 2026 录用

**核心贡献**：
提出首个原生全模态智能体 OmniAgent，将长视频理解建模为 POMDP 下的迭代式「观察-思考-行动」（OTA）循环。不同于传统「逐帧看完」的被动范式，模型主动选择需要感知的帧/音频/片段，将瞬态多模态感知压缩为持久文本记忆，使推理成本与视频长度解耦，可扩展至小时级视频。仅 7B 参数的 OmniAgent 在 LVBench 上以 50.5% 击败 10 倍大的 Qwen2.5-VL-72B（47.3%），在 10 个基准上取得开源 SOTA。该工作展示了 test-time scaling 的正向效应——更多推理轮次带来更强的性能。

**与AI Engineering的关联**：
为 Agent 架构设计提供了「主动感知 + 文本记忆压缩」的范本，对构建长上下文理解、多模态 RAG 和视频 Agent 系统有直接参考价值。其 agentic SFT + RL 训练流程也值得 LLM 应用开发者关注。

### 2. STARE：基于惊奇度加权的策略熵稳定性方法，解决 LLM 强化学习中的熵坍塌

**原标题**：STARE: Surprisal-Guided Token-Level Advantage Reweighting for Policy Entropy Stability
**作者/机构**：H. P. Luo 等
**链接**：[arXiv](http://arxiv.org/abs/2606.19236) · [GitHub](https://github.com/hp-luo/STARE)

**核心贡献**：
针对 GRPO 等 RL 方法在 LLM 推理训练中普遍出现的「策略熵坍塌」问题，该工作将 token 级熵动态分解为 advantage × surprisal 结构，识别出熵关键 token 的四象限分布及其近临界特性。提出 STARE 方法——惊奇度引导的 token 级 advantage 重加权 + 目标熵闭环门控，在数千步训练中维持熵稳定。在 AIME24/25 基准上，STARE 平均准确率超越 DAPO 等强基线 4–8%，且反思 token 和响应长度同步增长，实现持续的探索-利用平衡。

**与AI Engineering 的关联**：
直接解决 LLM 强化学习微调中的训练不稳定性问题，对做 RLHF/GRPO 训练管线的团队有极高实践价值。方法简单、代码开源，可快速集成到现有 RL 训练流程中。

### 3. 重新审视奖励监督：基于评分标准的条件自蒸馏

**原标题**：Rethinking Reward Supervision: Rubric-Conditioned Self-Distillation
**作者/机构**：Siyi Gu, Jialin Chen, Sophia Zhou, Arman Cohan, Rex Ying（Yale / Allen AI）
**链接**：[arXiv](http://arxiv.org/abs/2606.19327)

**核心贡献**：
针对 RL+可验证奖励训练中将反馈压缩为单一标量、损失细粒度改进信息的问题，提出「评分标准条件自蒸馏」框架。教师模型根据标准级别的评分细则（rubrics），对学生模型自身采样的推理轨迹提供 token 级指导，避免依赖单一参考答案。两阶段流程：(1) 学习生成任务特定评分标准，(2) 训练受评分标准引导的推理器。在多项科学推理基准上，平均超越 GRPO 1.0 分、超越 OPSD 0.9 分。

**与AI Engineering 的关联**：
为 LLM 推理能力的后训练提供了比纯 RL 更细粒度的监督范式，特别适合数学/代码/科学推理等有明确评分标准的场景。可与现有 GRPO 训练管线互补使用。

### 4. 用程序合成解释注意力机制

**原标题**：Explaining Attention with Program Synthesis
**作者/机构**：Amiri Hayes, Belinda Li, Jacob Andreas（MIT CSAIL）
**链接**：[arXiv](http://arxiv.org/abs/2606.19317)

**核心贡献**：
提出将 Transformer 注意力头近似为可执行的 Python 程序。方法：对给定注意力头，在随机样本上计算注意力矩阵，提示 LLM 生成候选程序，再在留出数据上按保真度重排序。仅用不到 1000 个生成的程序即可复现 GPT-2、TinyLlama-1.1B 和 Llama-3B 在 TinyStories 上的注意力模式，平均 IoU 超过 75%。替换 25% 的注意力头为程序化代理仅增加约 16% 的平均困惑度，下游 QA 性能基本保持。

**与AI Engineering 的关联**：
为 LLM 可解释性提供了可扩展的自动化 pipeline，程序化的注意力解释可直接用于模型调试、知识编辑和安全审计。对理解和控制大模型内部机制的前沿工程工作极具启发。

### 5. DreamReasoner-8B：面向扩散推理模型的分块课程学习

**原标题**：DreamReasoner-8B: Block-Size Curriculum Learning for Diffusion Reasoning Models
**作者/机构**：Zirui Wu（北大）, Lin Zheng, Jiacheng Ye, Shansan Gong, Xueliang Zhao, Yansong Feng, Wei Bi, Lingpeng Kong（港大）
**链接**：[arXiv](http://arxiv.org/abs/2606.19257) · [GitHub](https://github.com/DreamLM/DreamReasoner)

**核心贡献**：
探索块扩散语言模型是否能可靠地扩展用于长链式推理（long-CoT）。系统研究了训练和推理块大小对推理性能的影响，发现大块训练会严重损害推理能力。提出「块大小课程学习」——训练中逐步从细粒度过渡到粗粒度块，弥合了粒度差距。DreamReasoner-8B 是首个完全开源的 8B 块扩散推理模型，在数学和代码推理基准上与 Qwen3-8B 等主流自回归模型表现相当，为扩散语言模型在推理方向的应用提供了重要参考点。

**与AI Engineering 的关联**：
开辟了扩散模型作为自回归模型替代方案用于推理任务的可能性。块扩散天然支持并行解码，若推理性能持续追赶，可能在推理效率和延迟上有显著优势，值得推理部署工程师关注。
