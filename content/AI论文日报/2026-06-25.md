## 今日精选

### 1. OpenThoughts-Agent：Agent 模型训练数据配方完全开源
- **原标题**: OpenThoughts-Agent: Data Recipes for Agentic Models
- **作者/机构**: Negin Raoof, Richard Zhuang, Marianna Nezhurina 等（50 位作者，OpenThoughts 项目组）| [arXiv](http://arxiv.org/abs/2606.24855)
- **核心贡献**: ① 解决 Agent 模型训练数据缺乏公开配方的问题——目前开源方案（SWE-Smith、SERA、Nemotron-Terminal）只覆盖单基准；② 提出完全开源的数据筛选管线，通过 100+ 组消融实验系统性研究每个阶段，最终构建 10 万条训练数据；③ 在 Qwen3-32B 上微调后，7 个 Agent 基准平均准确率 44.8%，比最强开源 Agent 模型 Nemotron-Terminal-32B 提升 3.9 个百分点，且数据缩放性显著优于其他开源数据集
- **工程关联**: 直接服务于 Agent 模型训练——如果你在微调 Agent 模型，这套公开的数据配方和训练集是最直接的上游参考
- **价值点**: 首个全开源的 Agent 训练数据配方项目，训练数据/管线/模型全公开，可直接复现或用作文本 Agent 训练的起点

### 2. Qwen-AgentWorld：面向通用 Agent 的语言世界模型
- **原标题**: Qwen-AgentWorld: Language World Models for General Agents
- **作者/机构**: Yuxin Zuo, Zikai Xiao, Li Sheng 等（33 位作者，阿里通义千问团队）| [arXiv](http://arxiv.org/abs/2606.24597)
- **核心贡献**: ① 首次构建基于语言模型的 Agent 世界模型——Qwen-AgentWorld-35B 与 397B，覆盖 7 个领域，通过长链思维推理模拟 Agent 环境；② 三阶段训练管线（CPT→SFT→RL），利用 1000 万+ 真实环境交互轨迹；③ 作为解耦环境模拟器，支持数千个真实环境的可控仿真用于 Agent RL 训练，效果超越仅用真实环境的训练
- **工程关联**: 对 Agent RL 训练、环境模拟、世界模型构建有直接参考——可以用世界模型替代昂贵/不可控的真实环境进行 Agent 训练
- **价值点**: 首次将语言世界模型落地到 Agent 领域，开源的模型和代码可直接用于构建 Agent 训练环境模拟器，降低真实环境依赖成本

### 3. Agent 原生记忆系统：我们准备好了吗？
- **原标题**: Are We Ready For An Agent-Native Memory System?
- **作者/机构**: Wei Zhou, Xuanhe Zhou, Shaokun Han 等（8 位作者）| [arXiv](http://arxiv.org/abs/2606.24775)
- **核心贡献**: ① 指出当前 Agent 记忆评估只关注端到端任务指标（F1/BLEU），忽略了系统级成本、架构权衡和动态知识更新鲁棒性；② 提出四模块分析框架（表征存储/提取/检索路由/维护），系统评估 12 种代表性记忆系统在 5 个基准 11 个数据集上的表现；③ 关键发现：没有一种架构在所有场景下占优——有效性高度依赖记忆结构与工作负载瓶颈的对齐；局部维护比全局重组更具成本效率
- **工程关联**: 直接指导 Agent 记忆系统选型和架构设计——如果你的 Agent 需要跨会话持久记忆，这篇提供了最系统的工程决策参考
- **价值点**: 最全面的 Agent 记忆系统横向评测，结论「局部维护 > 全局重组」对降低 Agent 运行时成本有直接指导意义，代码开源

### 4. SHERLOC：为代码修复 Agent 量身打造的故障定位框架
- **原标题**: SHERLOC: Structured Diagnostic Localization for Code Repair Agents
- **作者/机构**: Hovhannes Tamoyan, Sean Narenthiran, Erik Arakelyan 等（5 位作者）| [arXiv](http://arxiv.org/abs/2606.24820)
- **核心贡献**: ① 解决 LLM Agent 在代码修复中一半推理预算浪费在故障定位上的问题；② 提出无训练的 SHERLOC 框架，将推理 LLM 与紧凑仓库工具和自恢复机制配对（无需微调或多 Agent 编排）；③ SWE-Bench Lite 准确率 84.33%@1，SWE-Bench Verified 召回率 81.27%@1；注入修复 Agent 后解决率平均提升 5.95pp，定位 token 减少 36.7%，总 token 减少 23.1%
- **工程关联**: 直接提升 SWE-bench 类代码修复 Agent 的效率——作为前端定位模块嵌入现有 Agent 流水线，显著降低 token 成本
- **价值点**: 无训练即插即用，直接提升代码修复 Agent 的解决率和效率，对 SWE-bench 刷榜和工程落地均有直接价值

### 5. DREAM：用自回归建模训练稠密检索嵌入
- **原标题**: DREAM: Dense Retrieval Embeddings via Autoregressive Modeling
- **作者/机构**: Yixuan Tang, Yi Yang（2 位作者）| [arXiv](http://arxiv.org/abs/2606.24667)
- **核心贡献**: ① 解决稠密检索器训练依赖昂贵标注数据（正负样本对）的问题；② 创新性地利用 LLM 的自回归 next-token 预测损失作为检索训练信号——通过将检索器生成的查询-文档相似度注入冻结 LLM 的指定注意力头，利用 LLM 预测损失的梯度训练检索器；③ 在 BEIR 和 RTEB 基准上，0.5B 到 3B 参数规模的嵌入模型均一致超越现有基线
- **工程关联**: 对 RAG 系统的检索模块训练有直接参考——无需人工标注的正负样本对即可训练高质量检索器，降低 RAG 系统构建成本
- **价值点**: 用 LLM 自带的自回归损失替代对比学习标注数据，大幅降低稠密检索器训练门槛，适合资源有限的 RAG 工程实践
