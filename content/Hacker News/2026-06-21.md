### ATProto 协议中不存在"实例"概念
- **原标题**：There are no instances in ATProto
- **类型**：技术讨论
- **热度**：⭐ 493 | 💬 271 评论
- **摘要**：Dan Abramov（React 核心开发者）深度解析了 AT Protocol（Bluesky 底层协议）的架构哲学。文章指出，"ATProto 中有几个实例？"本身是一个范畴错误——这是从 Mastodon/ActivityPub 继承来的"实例思维"。ATProto 将托管（PDS）与聚合（App/Relay）彻底分离，用户的身份和数据不属于任何单一服务器，像 RSS 时代的博客圈一样实现真正的去中心化。作者刚完成了从 Bluesky 官方 PDS 到 Eurosky 的迁移，体验几乎全自动。
- **链接**：[原文](https://overreacted.io/there-are-no-instances-in-atproto/)｜[HN 讨论](https://news.ycombinator.com/item?id=48599515)
- **价值点**：由顶级技术作者撰写的 ATProto 架构深度解析，澄清了社区对"去中心化社交"的普遍误解，是理解下一代社交协议设计的必读文章。

### 更大的模型不是出路：GPT-5.5 幻觉率是 GLM-5.2 的三倍
- **原标题**：GPT-5.5 hallucinates 3x more than MIT-licensed GLM-5.2
- **类型**：学术前沿
- **热度**：⭐ 395 | 💬 188 评论
- **摘要**：文章用硬数据论证了"模型越大越强"的范式正在撞墙。Z.ai 的 GLM-5.2（753B 参数，40B 激活）在 AA 智能指数上仅落后 GPT-5.5 4 分，但在 AA-Omniscient 幻觉基准上，GPT-5.5 的幻觉率高达 86%，DeepSeek V4 Pro 更是达到 94%，而 GLM-5.2 仅 28%。作者指出了 AI 的三难困境：原始能力、不确定性校准、计算效率——目前业界只关注第一项，忽视了后两者。
- **链接**：[原文](https://arrowtsx.dev/bigger-models/)｜[HN 讨论](https://news.ycombinator.com/item?id=48600167)
- **价值点**：以具体基准数据质疑"暴力 Scaling"路线，为 MoE/混合架构和模型校准能力提供了强有力的论据，直接关系到下一代模型的技术方向选择。

### 寻找屏幕无法显示的颜色：色彩科学的感官之旅

- **类型**：技术深度文章

- **链接**：[原文](https://moultano.wordpress.com/2026/06/19/where-to-find-the-colors-your-screen-cant-show-you/)｜[HN 讨论](https://news.ycombinator.com/item?id=48606140)
- **价值点**：将硬核色彩科学（CIE 色度图、三色视觉、同色异谱）与可感知的日常体验巧妙结合，让人重新审视我们习以为常的显示技术局限。

### 你能看见三棵树吗？欧洲城市绿化数据调查
- **原标题**：Can you see three trees?
- **类型**：行业观点
- **热度**：⭐ 256 | 💬 115 评论
- **摘要**：基于"3-30-300 规则"（从窗户能看见 3 棵树、社区 30% 树冠覆盖率、300 米内可达公园）的数据新闻项目。对欧洲 862 个城市的调查显示：仅 14% 的欧洲人满足全部三项标准，21% 一项都不满足。西班牙瓦伦西亚仅 10% 居民能看到三棵树。研究指出，达标社区的居民心理健康更好、用药更少，而树冠覆盖不足直接关联热浪死亡风险。
- **链接**：[原文](https://www.not-ship.com/can-you-see-three-trees/)｜[HN 讨论](https://news.ycombinator.com/item?id=48582374)
- **价值点**：将城市绿化这一公共政策议题转化为可量化、可验证的个人化指标，数据驱动的方式让读者能立即自查所在社区的绿化水平。

### 我把一个网站存进了 Favicon
- **原标题**：I Stored a Website in a Favicon
- **类型**：工具与项目
- **热度**：⭐ 248 | 💬 85 评论
- **摘要**：一个极客味十足的创意实验——将完整 HTML 网页编码为 Favicon 的像素数据。作者将 HTML 转为 UTF-8 字节，按 RGB 通道填入像素（每像素存 3 字节），最终将一个 208 字节的网页存入仅 9×9 像素的 PNG 图片中。浏览器端通过 Canvas API 读取像素、重建字节流、解码渲染——浏览器从自己的 Favicon 中"读"出了一个网站。
- **链接**：[原文](https://www.timwehrle.de/blog/i-stored-a-website-in-a-favicon/)｜[HN 讨论](https://news.ycombinator.com/item?id=48606619)
- **价值点**：虽不实用，但完美展示了"一切皆存储"的极客精神，巧妙运用 Canvas API 实现了一个自举的数据解码流程。

### 数据压缩详解：压缩即人工智能
- **原标题**：Data Compression Explained (2012)
- **类型**：技术深度文章
- **热度**：⭐ 172 | 💬 26 评论
- **摘要**：Matt Mahoney 的经典著作，系统阐述了数据压缩的理论基础：压缩 = 建模 + 编码。编码（哈夫曼、算术编码）是数学上已解决的问题，但建模（预测下一个符号的概率）被证明是 AI 完全问题——最优压缩等价于最优通用智能（Hutter 的 AIXI 理论）。文章涵盖了卡尔加里语料库基准、PAQ 系列的上下文混合技术，以及压缩率/速度/内存的不可能三角。
- **链接**：[原文](https://mattmahoney.net/dc/dce.html)｜[HN 讨论](https://news.ycombinator.com/item?id=48562662)
- **价值点**：压缩理论的经典教材，将信息论、算法与人工智能统一在一个框架下，深刻揭示了"压缩即理解"这一核心洞见。

### LLM 架构已经变得极其复杂
- **原标题**：LLMs Are Complicated Now
- **类型**：技术讨论
- **热度**：⭐ 113 | 💬 41 评论
- **摘要**：作者指出 LLM 架构正重蹈推荐系统的覆辙——从简洁的 Transformer 堆叠演变为包含分组查询注意力、压缩注意力、稀疏注意力、滑动窗口注意力、混合专家路由等多种变体的复杂系统。视觉和音频编码器从外挂变为内嵌，多 GPU 推理引入通信边界。核心矛盾：研究迭代需要灵活可组合的架构，但性能优化又要求深度定制融合内核。PyTorch FlexAttention 被视为解决此类问题的范例。
- **链接**：[原文](https://ianbarber.blog/2026/06/19/llms-are-complicated-now/)｜[HN 讨论](https://news.ycombinator.com/item?id=48605355)
- **价值点**：从推荐系统工程化历史视角审视 LLM 架构演进，提出了"可组合性优先于极致优化"的工程哲学，对模型开发者有实际指导意义。
