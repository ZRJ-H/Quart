#### Project Valhalla 详解：十年磨一剑，JDK 28 迎来值类型
- **原标题**：Project Valhalla, Explained: How a Decade of Work Arrives in JDK 28
- **热度**：⭐ 531 | 💬 325 评论
- **摘要**：Project Valhalla 是 Java 平台近十年来最重要的语言层面变革之一，核心是将"值的对象"（value objects）引入 JVM。它允许开发者创建没有对象标识、可以直接内联到堆栈和数组中的不可变类型，大幅减少内存开销和指针追踪，同时保持 Java 的类型安全。JDK 28 标志着 Valhalla 的正式落地，对高性能 Java 应用（金融、游戏、大数据）影响深远。HN 讨论极为热烈，有开发者称这是"Java 的重生"。
- **链接**：[原文](https://www.jvm-weekly.com/p/project-valhalla-explained-how-a)｜[HN 讨论](https://news.ycombinator.com/item?id=48595511)

#### AT 协议中没有"实例"这一概念
- **原标题**：There are no instances in ATProto
- **热度**：⭐ 318 | 💬 182 评论
- **摘要**：React 核心开发者 Dan Abramov 在个人博客深入分析了 Bluesky 背后的 AT Protocol 的去中心化架构。核心观点是 ATProto 不像 Mastodon/ActivityPub 那样以"实例"（instance）为基本单位，而是将身份、数据仓库和逻辑视图分离，形成一种更灵活的去中心化模型。理解这一设计对关心下一代社交协议架构的开发者至关重要。
