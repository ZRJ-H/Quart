# LLM Wiki 启动指南

## 在新Session中启动

### 方法1：直接说明

在新session中输入：

```
请阅读AGENTS.md，然后担任LLM Wiki维护专家角色，负责维护和更新我的知识库系统。
```

### 方法2：完整启动

```
请阅读AGENTS.md了解你的角色和行为规范，然后：
1. 读取wiki/index.md了解当前知识库状态
2. 读取wiki/log.md了解最近操作
3. 准备好处理我的查询或更新指令
```

### 方法3：快速启动

```
我是你的老板，你的角色是LLM Wiki维护专家。请先读取AGENTS.md和wiki/index.md，然后告诉我当前知识库状态。
```

## 注意事项

1. **每次新session都要读取AGENTS.md**，确保LLM了解自己的角色
2. **先读取index.md**，了解当前知识库状态
3. **先读取log.md**，了解最近操作，避免重复工作
4. **遵循Schema规范**，保持页面格式一致
5. **主动建立关联**，在实体/概念间创建交叉引用

---

## 故障排除

### 问题：LLM不了解自己的角色
**解决**：确保读取了AGENTS.md文件

### 问题：LLM不知道当前知识库状态
**解决**：先读取index.md和log.md

### 问题：LLM生成的页面格式不一致
**解决**：提醒LLM参考AGENTS.md中的页面格式规范

### 问题：LLM不知道如何处理新数据
**解决**：参考wiki/workflow-daily-update.md中的工作流程
