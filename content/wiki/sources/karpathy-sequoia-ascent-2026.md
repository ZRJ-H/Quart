# Sequoia Ascent 2026: Software 3.0, Agentic Engineering, and Jagged Intelligence

## 摘要
Karpathy 在 Sequoia Ascent 2026 上的炉边谈话（与合伙人 Stephanie Zhan 对谈）的文字稿和 AI 生成摘要。他提出 2025 年 12 月是 Agent 能力的拐点，编程的基本单元从"逐行输入代码"转变为"委托宏操作"。他定义了 Software 3.0 范式（通过 prompt、上下文、工具来编程 LLM），区分了 Vibe Coding（提升下限）和 Agentic Engineering（提升上限），并提出了 Jagged Intelligence 框架——模型能力尖峰取决于"可验证性 x 训练关注度 x 数据覆盖 x 经济价值"。文章还讨论了 MenuGen 案例、"动物 vs 幽灵"框架、Agent-Native 基础设施、以及教育中"你可以外包思考，但不能外包理解"的核心观点。

## 关键要点
- **2025 年 12 月 Agent 拐点**：Claude Code、Codex 等工具生成的代码块变得更大、更连贯、更可靠，编程转为委托式工作流
- **Software 3.0**：通过 prompt、上下文、工具、示例、记忆和指令来编程 LLM，上下文窗口成为主要杠杆
- **MenuGen 案例**：传统 Web 应用在 Software 3.0 下应消失，模型可直接从输入媒体变换到输出媒体
- **Verifiability 框架**："传统软件自动化你能指定的；LLM 和 RL 自动化你能验证的"
- **Jagged Intelligence**：模型能力尖峰 = 可验证性 x 训练关注度 x 数据覆盖 x 经济价值
- **Vibe Coding vs Agentic Engineering**：Vibe Coding 提升下限（让任何人做软件），Agentic Engineering 提升上限（专业协调 Agent 保持质量）
- **Animals vs Ghosts**：LLM 不是动物（没有生物驱动力、好奇心），而是人类造物的统计模拟
- **Agent-Native 基础设施**：未来产品需要为 Agent 而非仅为人类设计（机器可读模式、可粘贴指令、无头设置流）
- **教育资源**：关键句"你可以外包你的思考，但不能外包你的理解"
